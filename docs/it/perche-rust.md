# Perche' Rust

Rust non e' stato scelto per moda.

E' stato scelto perche' il problema lo giustifica e perche' il maintainer vuole
investire seriamente in questo linguaggio.

## Ragioni tecniche

Il dominio del progetto contiene continuamente:

- dati opzionali
- dati parziali
- fallback
- warning
- varianti di output
- JSON strutturato

Rust aiuta molto a rappresentare tutto questo senza scorciatoie deboli.

## Ragioni progettuali

Il clone non vuole fermarsi a una CLI monolitica.

Vuole avere:

- un core riusabile
- un modello dati pulito
- una base per una futura TUI

Rust e' molto adatto a questo tipo di progetto.

## Ragioni personali

Per Simone Di Maio questo progetto e' anche un modo concreto per spostare in
Rust una parte del proprio investimento tecnico futuro, dopo un percorso lungo
che parte da Pascal/Object Pascal e C/C++, passa per Delphi e arriva a Java come
linguaggio professionale principale.

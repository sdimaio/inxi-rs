# 0.1.0-alpha.1

Prima release pubblica alpha di `inxi-rs`.

`inxi-rs` è una reimplementazione indipendente in Rust ispirata a
[`inxi`](https://github.com/smxi/inxi). Questa release è focalizzata su un core
Linux `read-only`, prudente dal punto di vista operativo, con output
deterministico `screen` e `json`.

## Punti principali

- libreria riusabile `inxi-core` con frontend separato `inxi-cli`
- collector Linux per `System`, `Machine`, `CPU`, `Memory`, `Graphics`,
  `Network`, `Drives`, `Partitions`, `Swap` e `Info`
- filtro privacy con `-z`
- modalità `--self-check` per diagnosticare collector, warning e fallback
- test con fixture parser e golden test sull'output
- documentazione di progetto in inglese e in italiano

## Modello di sicurezza

`inxi-rs` è progettato per restare osservativo:

- nessuna scrittura sul filesystem
- nessuna scrittura su `/proc`, `/sys` o `/dev`
- nessuna esecuzione di shell
- nessuna escalation di privilegi
- nessun accesso di rete dal tool
- comandi esterni ammessi solo se auditati e con argomenti fissi

## Note di perimetro

- la piattaforma supportata al momento è solo Linux
- il progetto è ancora `alpha`
- la parità completa con `inxi` upstream non è ancora un obiettivo
- la TUI è prevista architetturalmente, ma non è inclusa in questa release

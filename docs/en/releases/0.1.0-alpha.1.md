# 0.1.0-alpha.1

Initial public alpha for `inxi-rs`.

`inxi-rs` is an independent Rust reimplementation inspired by
[`inxi`](https://github.com/smxi/inxi). The current release focuses on a
safety-first, read-only Linux core with deterministic `screen` and `json`
output.

## Highlights

- reusable `inxi-core` library with a separate `inxi-cli` frontend
- Linux collectors for `System`, `Machine`, `CPU`, `Memory`, `Graphics`,
  `Network`, `Drives`, `Partitions`, `Swap`, and `Info`
- privacy filtering via `-z`
- `--self-check` mode for collector diagnostics and fallback visibility
- parser fixture tests and golden output tests
- English and Italian project documentation

## Safety Model

`inxi-rs` is designed to remain observational:

- no filesystem writes
- no writes to `/proc`, `/sys`, or `/dev`
- no shell execution
- no privilege escalation
- no network access from the tool itself
- external commands allowed only when audited and fixed-argument

## Scope Notes

- platform support is currently Linux-only
- the project is still `alpha`
- full feature parity with upstream `inxi` is not yet a goal
- TUI support is planned architecturally, not delivered in this release
